# Drafting Workflow

本参考文件只服务于最终 GUI 外观设计专利 Word 成稿，不再服务于分析报告型工作流。

## When To Use

在这些场景读取本文件：

- 已经确定最终保留的状态图集合
- 已经开始整理可直接填入模板的正文
- 需要起草摘要、各界面状态说明、交互说明
- 需要把总图识别结果转成最终单张状态图成稿流程

如果用户还没确定保留哪些状态图，先回到主 `SKILL.md` 做状态收束；不要先铺案件策略、风险提示或视图计划。

## Step A: 先收束最终状态图

先确认：

1. 最终保留多少张状态图
2. 哪些状态属于主流程
3. 哪些状态要删除
4. 是否有用户明确要求删除的步骤

如果删除某一步，例如注册页：

- 删除该步骤标题
- 删除该步骤说明
- 删除该步骤图片
- 删除对应 JPG 文件
- 后续状态图从 `变化状态图01` 重新连续编号

## Step B: 总图只做结构识别

如果用户上传的是总图：

- 先识别子图数量
- 按版面顺序列出候选状态
- 确认哪些子图最终保留
- 进入最终成稿前，要求补齐每个保留状态对应的单张图

总图不能直接作为最终 Word 插图使用。

## Step C: 单张状态图才用于最终成稿

只有拿到单张状态图后，才进入正文起草和最终导出。

最终 Word 和 JPG 图片包中使用的都必须是单张状态图，而不是拼接总图。

## Step D: 正文只写模板需要的内容

默认只起草这些部分：

- `摘要`
- `各界面状态说明`
- `交互说明`

不要默认新增这些一级栏目：

- `案件摘要`
- `风险提示`
- `申请策略`
- `视图计划`
- `可申请性分析`

## Step E: 各界面状态说明写法

逐图说明只写截图可见事实，按这个顺序：

1. 当前界面处于什么状态
2. 主要区域有哪些
3. 区域内有哪些可见元素
4. 当前界面可执行什么操作

避免写入：

- 主观评价词
- 技术实现
- 算法逻辑
- 后台逻辑
- 营销语言

推荐输出模板：

```markdown
## 变化状态图01 · [状态名]
[一段客观界面描述]
```

## Step F: 交互说明写法

交互说明用于把最终保留状态图串起来，**必须包含三层结构**，缺一层都会削弱专利保护力。

### 三层结构

| 层 | 写什么 | 关键句式 |
| --- | --- | --- |
| ① 切换方式 | 各界面如何在用户操作下动态切换（滚动 / 点击 / 输入 / 模式切换） | "通过…操作依次展示" / "可在用户不同操作下动态切换" |
| ② 保护要点（**白名单**） | 主张哪些**抽象的视觉编排**受保护 | **"保护要点在于…的组合布局 / 视觉结构 / 编排关系 / 呼应关系"** |
| ③ 不涉及部分（**黑名单**） | 主动放弃对**可变内容**的主张 | **"不涉及的部分：文字内容本身、品牌LOGO本身、图片素材、具体数据字段、网页背景色、动画效果"** |

为什么要写第②③层：

- 只写第①层 → 容易被审查员理解为"专利保护的是文字 + Logo + 图片"，反而被驳
- 写了第②层 → 把保护范围抽象到"视觉编排"，符合 GUI 外观设计专利的客体本质
- 写了第③层 → 主动放弃可变内容，未来网站改文案、换 Logo、改数据、换客户图、改背景色，专利依然有效

### 标准句式模板

```text
交互说明：
各界面[通过…操作 / 在用户不同操作下]依次展示 / 动态切换，
[一句承接：不同色块区域代表… / 不同模式代表… / 各界面按图1-图N状态分别展示]。
保护要点在于[列举 2-4 项抽象视觉编排，如 "整体多色分区模块布局"、"左侧任务树与右侧工作区的组合布局"、
"图文左右交替编排"、"顶部导航与底部表单的呼应结构"、"独特的多状态界面展示" 等]。

不涉及的部分：文字内容本身、品牌LOGO本身、图片素材、具体数据字段内容、网页背景色、动画效果。
```

### 示例 1（网页 / 官网类）

> 交互说明：
> 各界面通过用户滚动操作依次展示，不同色块区域代表页面的功能层次。主界面至核心服务界面的过渡通过渐变背景与模块色块自然衔接。**保护要点在于整体多色分区模块布局与品牌视觉结构**，不涉及的部分包括：**文字内容本身、LOGO本身、图片素材、具体数据字段及动画效果**。

### 示例 2（多页应用 / 工作台类）

> 交互说明：
> 各界面可在用户不同操作下动态切换，**保护点在于左侧品牌/任务树与右侧动态工作区的组合布局及独特的多状态界面展示**，上述界面按图1-图6状态分别展示。
> **不涉及的部分：文字内容本身、品牌LOGO本身、具体数据字段内容、网页背景色。**

### 黑名单常见项（按场景灵活选取）

- 文字内容本身（标题、副标题、按钮文字、说明段）
- 品牌 LOGO 本身、客户标识本身、第三方商标
- 图片素材本身（Hero 配图、案例配图、装饰插画）
- 具体数据字段内容、图表数值与刻度
- 表单字段名、占位符
- 网页背景色值、主题色值
- 动画效果、过渡效果、加载效果
- 字体本身、字号、字重

### 不允许写入交互说明的内容

- 后台逻辑、算法判断、性能优势
- 营销语言（"行业领先"、"独家创新"）
- 技术实现细节（API、数据库、中间件）
- 用户角色 / 权限差异

## Step G: 进入最终 Word 导出

当以下条件满足时，直接进入最终 Word：

1. 最终保留状态图集合已确定
2. 单张状态图已齐
3. 摘要、状态说明、交互说明已具备
4. 模板关键字段已具备或允许留空/待补充

不要再把文档导出视为“可选交付”。  
对于这个 skill，最终 Word 和 JPG 图片包就是默认交付。
