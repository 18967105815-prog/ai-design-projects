# China GUI Design Patent Rules

本文件用于给 `gui-design-patent-cn` 提供当前中国 GUI 外观设计专利工作流的规则底座。它不是法条全文，而是按办案流程整理的操作摘要。

## Source Priority

按这个顺序引用和判断：

1. 国家知识产权局《涉及图形用户界面的产品外观设计专利申请指引》
2. 国家知识产权局《专利审查指南（2023）》第一部分第三章第 4.5 节
3. 《中华人民共和国专利法实施细则（2023年修订）》第三十条、第三十一条、第四十条
4. 国家知识产权局对《专利审查指南（2023）》修改解读（2024-01-18）

## Key Working Rules

### 1. 命名规则

- 产品名称应写明 GUI 的具体用途和其所应用的产品。
- 名称一般应包含“图形用户界面”字样。
- 动态 GUI 名称应包含“动态”字样。
- 不要笼统写成“软件图形用户界面”“操作图形用户界面”。
- 若申请 GUI 中的局部，产品名称还应写明请求保护的局部。

### 2. 简要说明规则

- 应写明产品名称、用途、设计要点。
- 应指定最能表明设计要点的图片或照片。
- 省略视图、请求保护色彩、局部保护、基本设计等事项，应在简要说明中说明。
- 不得使用商业性宣传用语，不得说明产品性能。
- 对 GUI 案件，必要时说明 GUI 所在区域、人机交互方式及变化过程。

### 3. 视图规则

- 设计要点同时包含 GUI 与产品外形时，按一般整体外观设计规则准备视图。
- 设计要点仅在 GUI 时，至少提交 GUI 所涉及面的产品正投影视图；必要时补 GUI 视图。
- 局部设计要体现请求保护部分；如需表达位置和比例关系，可带所应用产品。
- 请求保护色彩时，应提交彩色图片或照片。

### 4. 动态 GUI 规则

- 主视图应为起始状态所涉及面的视图。
- 变化状态图应按动态变化先后顺序标注。
- 所提交状态应能唯一确定完整变化过程。
- 官方认为必要时，可要求提交表明变化过程的视频类文件。

### 5. 相似设计与组案规则

- 同一产品的多项相似外观设计可作为一件申请提出。
- 简要说明中应指定一项作为基本设计。
- 一件外观设计申请中的相似外观设计通常不得超过 10 项。

### 6. 风险筛查提示

- 与人机交互无关的显示图案、纯壁纸、纯开关机画面，应列为高风险项。
- 游戏界面等争议场景，不直接下可授权结论，改为“需代理人复核”。
- 已公开、将上线、参展、宣传投放、灰度测试等，都应作为新颖性风险节点单列。

## Practical Workflow Mapping

将规则映射到案件流程时，优先这样处理：

1. 先定产品命名与法域
2. 再定整体 / 局部
3. 再定静态 / 动态
4. 再定是否带所应用产品
5. 再定是否需要相似设计组案
6. 最后写视图计划和简要说明
