# 标准 GUI 交底 Word 模板规则

这个 skill 默认按标准 `02界面说明.docx` 结构导出最终 GUI 外观专利 Word。

用户没有另外提供模板时，直接使用 skill 自带的 `assets/word-template.docx`，并沿用下面同样的分页和正文结构。

用户提供的 `02界面说明.docx` 示例，只是帮助确认这套标准结构，不代表每次都要等用户再上传模板文件。

## 固定分页

- page01：只保留封面标题 `专利申请GUI交底说明` 和蓝字提示 `（图片需单独提供JPG）`。
- page02：只保留信息表，包括日期、专利类型、领域、专利名称、申请人、发明人/身份证号、联系方式、固定电话、手机、邮箱等原字段。
- page03：从 `界面说明如下：` 开始正文。

不要把信息表、摘要或状态图挤到 page01。不要让 `界面说明如下：` 出现在 page01 或 page02。

## 图片输入规则

最终 Word 成稿必须使用单张状态图。

- 总拼图 / 总流程图：只用于识别状态集合和流程关系
- 单张状态图：用于插入 Word 正文和导出 JPG 图片包

如果用户当前只有总图，必须在进入最终 Word 生成前补齐单张状态图，或明确从总图中拆出每个最终保留状态的单图。

没有单张状态图时，不应直接交付最终 Word。

## 正文栏目

正文按模板原顺序：

1. `界面说明如下：`
2. `摘要：`
3. 摘要段落和项目符号列表
4. 设计要点段落
5. `各界面状态说明：`
6. 状态标题，例如 `申请媒体账户界面（变化状态图01）`
7. 状态说明段落
8. 对应图片
9. `交互说明：`
10. 一段连续交互说明

不要新增模板外一级标题，例如“案件摘要”“风险提示”“申请策略”“视图计划”“图面处理要求”。

## 删除或重排状态

用户要求删除某个步骤时：

- 删除该步骤标题、说明、图片和单独 JPG。
- 后续状态图从 `变化状态图01` 开始重新连续编号。
- 交互说明同步删除该步骤，不得残留相关词。
- 检查正文中不得残留被删状态名称。

例如删除注册页后，正文可从：

- `申请媒体账户界面（变化状态图01）`
- `认证企业信息界面（变化状态图02）`
- `申请TikTok开户界面（变化状态图03）`
- `激活账号界面（变化状态图04）`
- `钱包充值界面（变化状态图05）`

开始，不再出现 `账号注册界面`。

## 脚本化导出（推荐）

为避免手工粘贴导致的分页错位、样式丢失或示例文案残留，最终 `02界面说明.docx` 应优先使用仓库脚本生成：

- `scripts/export_gui_patent_docx.py`：读取结构化 JSON + 模板 DOCX，清空锚点 `界面说明如下：` 之后的正文并重写摘要 / 列表项 / 逐图说明 / 插图 / 交互说明，同时在输出目录生成 `jpg/` JPEG 图片包。
- 模板必须使用与公司实例相同的骨架：**封面页 → 分页 → 信息表 → 分页 → `界面说明如下：`**.
- JSON 字段与示例见 [intake-and-output-schema.md](intake-and-output-schema.md)。

脚本导出后仍应人工快速目视检查下列 OOXML/版式要点。

## OOXML/生成检查

生成 DOCX 后至少检查：

- 标题页后是否仍有 page break。
- 信息表后是否仍有下一页分节或等效分页。
- `界面说明如下：` 是否从第三页开始；必要时给该段落设置 `pageBreakBefore`。
- Word 内是否全部使用单张状态图。
- 模板原案例文案是否清除。
- 删除状态和空项目符号是否清除。
- Word 内图片数量和 JPG 包图片数量是否与状态说明一致。
